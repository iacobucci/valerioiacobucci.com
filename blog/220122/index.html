Use Chrome from Gnome shell search

or any other browser really...

Gnome shell is a very cool enviroment but it is not always easy to customize. I like the integrated search feature, that indexes .desktop shortcuts and files from the home user directory, but i'd like to browse the web with it too. I mean, you could technically do that out of the box but you'd be stuck with Epiphany (i.e. Gnome Web), which is pretty unusable.
Some time ago i've managed to port this extension so that it would use DuckDuckGo, and posted it on Github, but due to the recent Gnome's updates, it doesnt work anymore. And i could barely understand what the extension did in the first place, so i don't think to pick it up anytime soon.
Still, i've found a quirky workaround that lets me do what i need: type a query into the search box and have my browser of choice manage requests and stuff, though it doesnt provide suggestions.



At this point i've got to specify that i use arch btw... So this guide will suggest you to use pacman hooks to make the changes persistent after each and every update. The procedure consists into tricking gnome to execute Chrome (or any other browser) when calling for the epiphany binary. All you need to do really is:

sudo mv /usr/bin/epiphany /usr/bin/epiphany-old
sudo ln -sn /usr/bin/google-chrome-stable /usr/bin/epiphany

And in order to apply the correct icon:

for i in $(/usr/bin/ls -d /usr/share/icons/Papirus/*/);
do
sudo mv ${i}apps/epiphany.svg ${i}apps/epiphany.old
sudo ln -sn ${i}apps/google-chrome.svg ${i}apps/epiphany.svg
done
As i said before, in order to automate this process on package updates you can use this hook and this hook.
And it should be all done, now you can go on with your nasty Chrome searches directly from the Gnome shell search!
